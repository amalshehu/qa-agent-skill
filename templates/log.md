# QA Wiki log

Append-only. One line per run: `## [YYYY-MM-DD] run | tracker=<x> | N found (M new, K regressions)`

Optionally, an indented note under that line for anything about the run's *mechanics* worth flagging for next time (a flaky selector, an undocumented login step, an escalation limit that got hit) — this is the wiki learning about itself, not just about the app.
